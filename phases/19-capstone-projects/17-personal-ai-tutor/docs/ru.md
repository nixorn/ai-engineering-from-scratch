# Капстоун 17 — Персональный AI-репетитор (адаптивный, мультимодальный, с памятью)

> Khanmigo (Khan Academy), Duolingo Max, Google LearnLM / Gemini for Education, Quizlet Q-Chat и Synthesis Tutor к 2026 году запустили адаптивное мультимодальное обучение в масштабе. Общая форма: политика в стиле Сократа (никогда просто не выдавать ответ), модель ученика, обновляющаяся после каждого взаимодействия (в стиле Bayesian Knowledge Tracing), ввод голосом + текстом + фото математических задач, извлечение по графу учебной программы, планирование интервальных повторений и жёсткие фильтры безопасности для возрастно-подходящего контента. Цель капстоуна: выпустить предметно-ориентированного репетитора (алгебра K-12 или вводный Python), провести двухнедельное исследование эффективности с 10 учащимися и пройти аудит безопасности контента.

**Тип:** Капстоун  
**Языки:** Python (бэкенд, модель ученика), TypeScript (веб-приложение), SQL (граф учебной программы через Postgres + Neo4j)  
**Пререквизиты:** Фаза 5 (NLP), Фаза 6 (речь), Фаза 11 (LLM engineering), Фаза 12 (мультимодальность), Фаза 14 (агенты), Фаза 17 (инфраструктура), Фаза 18 (безопасность)  
**Задействованные фазы:** P5 · P6 · P11 · P12 · P14 · P17 · P18  
**Время:** 30 часов

## Проблема

Адаптивное обучение раньше было нишей исследований в ed-tech. К 2026 году это потребительский продукт. Khanmigo развёрнут в большинстве школьных округов США. Duolingo Max достиг десятков миллионов MAU. LearnLM / Gemini for Education от Google обеспечивает тьюторинг в Google Classroom. Quizlet Q-Chat работает рядом с карточками. Synthesis Tutor стал вирусным как «репетитор для любознательных детей». Общие элементы: мультимодальный ввод (печатать, говорить, фотографировать уравнения), сократическая педагогика (сначала спросить, потом объяснить), модель ученика, обновляющаяся после каждого взаимодействия, и строгая возрастная безопасность.

Вы создадите один из таких продуктов для конкретной аудитории. Планка измерения — реальное исследование эффективности: результаты pre-test и post-test за две недели на 10 учащихся. Голосовой цикл должен ощущаться естественно (сабстек капстоуна 03). Память должна уважать приватность. Фильтр безопасности должен пройти red-team с учётом COPPA для K-12.

## Концепция

Четыре компонента. **Политика репетитора** — сократический цикл: когда ученик просит ответ, политика задаёт наводящий вопрос; когда ученик отвечает правильно, система переходит к следующей концепции; когда ученик застрял, система даёт поэтапную подсказку. **Модель ученика** — Bayesian Knowledge Tracing (или простой вариант), которая после каждого взаимодействия обновляет вероятность освоения для каждого узла учебной программы. **Граф учебной программы** — Neo4j-граф концепций с рёбрами пререквизитов; политика проходит по графу, чтобы выбрать следующую концепцию. **Память** — эпизодическое + семантическое хранилище (в стиле agentmemory), где сохраняются прошлые взаимодействия, ошибки и предпочтения.

UX мультимодален. Текстовый ввод для печатных ответов. Голосовой ввод через LiveKit + Whisper (переиспользуйте капстоун 03). Фото-ввод математических задач через dots.ocr или PaliGemma 2. Голосовой вывод через Cartesia Sonic-2. Безопасность обеспечивается Llama Guard 4 плюс возрастно-подходящий фильтр (блокирует взрослый контент, насилие, самоповреждение) и политика хранения памяти с учётом COPPA.

Исследование эффективности — главный артефакт. 10 учащихся, pre-test и post-test, две недели. Отчитайте дельту прироста обучения и доверительный интервал. Сравните с неадаптивным baseline (тот же контент, поданный линейно без политики репетитора).

## Архитектура

```
устройство ученика
  |
  +-- текст        -> веб-приложение
  +-- голос        -> LiveKit Agents (ASR + TTS)
  +-- фото-математика -> dots.ocr / PaliGemma 2
       |
       v
  политика репетитора (LangGraph)
       - сократическая decision head
       - выбор следующей концепции (обход графа учебной программы)
       - генератор поэтапных подсказок
       - обновление освоения
       |
       v
  модель ученика (BKT / item-response theory)
       - вероятность освоения по каждой концепции
       - планировщик интервальных повторений (SM-2 или FSRS)
       |
       v
  память (в стиле agentmemory)
       - эпизодическая: каждое взаимодействие
       - семантическая: усвоенные ошибки, предпочтения
       - политика хранения: с учётом COPPA / GDPR
       |
       v
  граф учебной программы (Neo4j)
       - рёбра пререквизитов
       - прикреплённый OER-контент
       |
       v
  безопасность:
    Llama Guard 4 + возрастно-подходящий фильтр
    доступ к памяти ограничен скоупом learner ID
```

## Стек

- Выбор предмета: алгебра K-12 или вводный Python (выберите один для глубины)
- Политика репетитора: LangGraph поверх Claude Sonnet 4.7 (с prompt caching)
- Модель ученика: классический Bayesian Knowledge Tracing или FSRS для интервалов
- Граф учебной программы: Neo4j концепций + рёбра пререквизитов + OER-контент
- Память: персистентное векторное + эпизодическое + семантическое хранилище в стиле agentmemory
- Голос: LiveKit Agents 1.0 + Cartesia Sonic-2 (переиспользуйте сабстек капстоуна 03)
- Фото-математика: dots.ocr или PaliGemma 2 для распознавания уравнений
- Безопасность: Llama Guard 4 + кастомный возрастно-подходящий фильтр
- Оценка: генерация вопросов по уровням Bloom, harness pre/post тестов, инструменты исследования эффективности

## Реализуйте

1. **Граф учебной программы.** Постройте Neo4j-граф из 50-150 узлов концепций (например, алгебра K-12 от «числовой прямой» до «квадратичной формулы») с рёбрами пререквизитов. Прикрепите OER-контент к каждому узлу (Open Textbook, OpenStax).

2. **Модель ученика.** Инициализируйте Bayesian Knowledge Tracing с априорными параметрами: guess, slip, learn-rate. Обновляйте освоение по каждой концепции после каждого взаимодействия. Сохраняйте данные отдельно для каждого ученика.

3. **Политика репетитора.** LangGraph с узлами: `read_signal` (ответ ученика правильный / частичный / застрял?), `select_concept` (обход графа учебной программы и выбор концепции с наивысшим приоритетом), `scaffold` (сократический промпт), `update_mastery`.

4. **Память.** Каждое взаимодействие записывается в эпизодическое хранилище. Ошибки и предпочтения продвигаются в семантическую память. Политика хранения с учётом COPPA: автоудаление через 1 год, доступ для родителей.

5. **Голосовой путь.** Воркер LiveKit Agents подключается к политике репетитора. ASR через Whisper-v3-turbo. TTS через Cartesia Sonic-2. Поддержка barge-in (переиспользуйте механику капстоуна 03).

6. **Путь фото-математики.** Загрузка или захват изображения; запуск dots.ocr или PaliGemma 2 для распознавания уравнения; передача репетитору как структурированного ввода.

7. **Безопасность.** Каждый вывод модели проходит через Llama Guard 4 + возрастно-подходящий фильтр (блокирует самоповреждение, взрослый контент, насилие). Доступ к памяти ограничен learner ID; предусмотрите родительский интерфейс для удаления данных.

8. **Исследование эффективности.** 10 учащихся, pre-test (стандартизированная базовая проверка на 30 вопросов), две недели взаимодействия с репетитором (3 сессии в неделю), post-test. Сравните с неадаптивной baseline-группой из 10 учащихся на том же контенте.

9. **Еженедельные отчёты о прогрессе.** Для каждого ученика автоматически генерируйте PDF-сводку с пройденными темами, траекториями освоения и рекомендованными следующими шагами.

## Используйте

```
ученик: "Я не понимаю, почему 3x + 6 = 12 означает x = 2"
[signal]   застрял
[concept]  'изоляция переменных' (пререквизит: addition-subtraction-equality)

[scaffold] "отлично. теперь чему равно 3x / 3?"
```

## Упражнения

1. Проведите исследование эффективности с адаптивной моделью ученика и без неё (случайный порядок концепций). Отчитайте дельту. Ожидается, что адаптивная версия выиграет, но интересен именно размер эффекта.

2. Добавьте мультимодальный эксперимент: тот же вопрос по концепции в текстовом, голосовом и фото-формате. Измерьте, сходятся ли учащиеся быстрее в предпочитаемой ими модальности.

3. Создайте родительский дашборд: практиковавшиеся темы, траектории освоения, предстоящие концепции, события безопасности (срабатывания guardrail). В соответствии с COPPA.

4. Добавьте режим переключения языка: репетитор принимает ввод на испанском и обучает на испанском. Измерьте покрытие X-Guard.

5. Проведите стресс-тест приватности памяти: убедитесь, что ученик A не может видеть данные ученика B даже через атаку повторной загрузкой голосового клипа. Логируйте попытку доступа и отправляйте алерт.

## Ключевые термины

| Термин | Как обычно говорят | Что это на самом деле означает |
|------|---------------------|--------------------------------|
| Socratic policy | "Сначала спроси, не выдавай ответ" | Репетитор задаёт наводящий вопрос, а не даёт готовый ответ |
| Bayesian knowledge tracing | "BKT" | Классические уравнения модели ученика для вероятности освоения по каждой концепции |
| FSRS | "Free Spaced Repetition Scheduler" | Планировщик интервальных повторений 2024 года, лучше SM-2 |
| Curriculum graph | "Concept DAG" | Neo4j-граф концепций с рёбрами пререквизитов |
| Episodic memory | "Лог каждого взаимодействия" | Каждое взаимодействие сохраняется для последующего извлечения |
| Semantic memory | "Хранилище усвоенных паттернов" | Сжатые ошибки и предпочтения, повышенные из эпизодической памяти |
| COPPA | "Закон о приватности детей" | Закон США, ограничивающий сбор данных о детях младше 13 лет |
